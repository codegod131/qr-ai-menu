# Product / QA / Ops Actionable Tasks
**Owner:** Member 4 (Role D)
**Stack:** Vercel, Render/Railway, GitHub/GitLab

## Day 1: Foundations
- [ ] Set up Git repository and basic CI pipeline.
- [ ] Choose deployment targets (e.g., Vercel for Frontend, Render for Backend).
- [ ] Write realistic seed datasets for two test verticals:
  - 10-15 menu items with rich descriptions (e.g., taste, mood, spice level, dietary tags).
  - 10-15 footwear items with rich descriptions (e.g., outfit style, use-case, season).
- [ ] Define the 4-5 core success metrics to track.

## Day 2: Core Loop
- [ ] Deploy the application to a staging environment.
- [ ] Run both seed datasets through the live flow (insert via Client Portal).
- [ ] Start writing manual QA test cases.
- [ ] Ensure event logging is instrumented correctly (query text, results shown, which result user taps).

## Day 3: Measure + Learn + Polish
- [ ] Conduct full team QA pass on both verticals with real seed data; report bugs.
- [ ] Recruit 5-10 people (colleagues/friends) to scan the QR on their phones and try real queries.
- [ ] Capture user feedback: Did they find what they wanted? How long did it take? Did they rephrase?
- [ ] Prepare metrics for the Build-Measure-Learn review.
- [ ] Prepare a short demo script + 3-5 slide pitch summarizing what was learned.
